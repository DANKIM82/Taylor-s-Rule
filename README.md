# 📊 Taylor's Rule Macro Dashboard

미국 매크로 데이터(FRED)를 매일 자동 수집하여 **테일러 준칙(Taylor's Rule)** 과 **실질금리-성장률 격차(r-g)** 를 계산하고, 현재 통화정책이 긴축적인지 완화적인지를 판단하는 퀀트 매크로 모니터링 도구입니다.

콘솔에 요약 대시보드를 출력하고, 전체 시계열을 포맷팅된 엑셀 파일(`Macro_Daily_DB.xlsx`)로 백업합니다.

---

## 🧮 1. 이론 배경

### 테일러 준칙 (Taylor's Rule)

테일러 준칙은 1993년 경제학자 **John B. Taylor**가 제안한, 중앙은행의 적정 정책금리를 산출하는 공식입니다. 핵심 아이디어는 *"중앙은행은 인플레이션과 경기(생산갭)에 반응하여 금리를 조정해야 한다"* 는 것입니다.

기본 형태는 다음과 같습니다.

```
i = r* + π + 0.5 × (π − π*) + 0.5 × (산출갭)
```

| 기호 | 의미 |
|------|------|
| `i`  | 모델이 제시하는 적정 명목 정책금리 |
| `r*` | 중립 실질금리 (경제의 잠재 성장 추세) |
| `π`  | 현재 인플레이션율 |
| `π*` | 목표 인플레이션 (본 코드에서는 **2.0%**) |
| `(π − π*)` | 인플레이션 갭 → 물가가 목표보다 높으면 금리 인상 압력 |
| `산출갭` | 실질 GDP가 잠재 GDP에서 벗어난 정도 → 경기 과열이면 금리 인상 압력 |

> **본 코드의 구현**: 중립 실질금리 `r*` 대용치로 **5년 평균 실질 GDP 성장률**을 사용합니다.
> ```
> Taylor_Target_Rate = GDP_5Y_Avg + Inflation + 0.5×(Inflation − 2.0) + 0.5×GDP_Gap
> ```

**해석**
- `Taylor_Gap = 실제 기준금리 − 모델 적정금리 > 0` → **준칙 대비 긴축** (연준이 경제 체력 대비 금리를 높게 유지)
- `Taylor_Gap < 0` → **준칙 대비 완화** (지표 대비 통화정책이 느슨함)

### 실질금리 − 성장률 격차 (r − g)

금융 환경의 긴축/완화 정도를 보는 또 다른 프레임입니다.

```
r = 10년물 국채금리 − 인플레이션 (시장 실질금리)
g = 5년 평균 실질 GDP 성장률
Financial_Stance = r − g
```

- `r − g > 0` → **긴축 국면**: 자금 조달 비용이 경제 성장 탄력보다 높음
- `r − g < 0` → **완화 국면**: 금융 환경이 완화적이며 투자가 장려되는 구간

## 📥 2. 데이터 소스 (FRED)

[FRED](https://fred.stlouisfed.org/) (St. Louis Fed)에서 `pandas_datareader`로 자동 수집합니다. 수집 시작일은 `2000-01-01`입니다.

| 티커 | 설명 | 빈도 |
|------|------|------|
| `DFF`      | 유효 연방기금금리 (Effective Fed Funds Rate) | 일간 |
| `DGS10`    | 미 국채 10년물 금리 | 일간 |
| `CPIAUCSL` | 소비자물가지수 (CPI) | 월간 |
| `GDPC1`    | 실질 GDP | 분기 |
| `GDPPOT`   | 잠재 GDP (Potential GDP) | 분기 |

월간·분기 데이터는 **Forward Fill (`ffill`)** 로 일간 기준에 맞춰 채워, 새 발표 전까지 직전 값을 유지합니다.

### 계산되는 파생 지표

| 컬럼명 | 산식 |
|--------|------|
| `Inflation_YoY(%)`       | CPI 전년동월비 |
| `GDP_Growth_YoY(%)`      | 실질 GDP 전년동기비 |
| `GDP_Growth_5Y_Avg(%)`   | GDP 성장률 5년(20분기) 이동평균 → `g` |
| `GDP_Gap(%)`             | (실질 GDP − 잠재 GDP) / 잠재 GDP |
| `Real_Rate_10Y(%)`       | 10년물 금리 − 인플레이션 → `r` |
| `Financial_Stance(r-g)`  | `r − g` |
| `Taylor_Target_Rate(%)`  | 테일러 준칙 적정금리 |
| `Taylor_Gap(%)`          | 실제 기준금리 − 적정금리 |

## 🏗️ 3. 코드 구조

[Taylor Rule.py](Taylor%20Rule.py) 는 4개의 모듈식 엔진으로 구성됩니다.

```
Taylor Rule.py
│
├── 1. fetch_macro_data()      # 데이터 수집 & 연산 엔진
│      └ FRED 수집 → 파생지표 계산 → 일간 병합/ffill → 최신순 정렬된 DataFrame 반환
│
├── 2. print_dashboard(df)     # 콘솔 출력 엔진
│      └ 최신일 기준 요약 + (r-g) / 테일러 준칙 두 가지 분석 결론 출력
│
├── 3. update_excel_db(df)     # 엑셀 출력 엔진
│      └ output/ 폴더 생성 → 포맷팅된 .xlsx 저장 (틀고정, 열너비, 소수점 2자리)
│
└── 4. __main__               # 메인 실행 블록 (수집 → 대시보드 → 엑셀 저장, 예외 처리 포함)
```

엑셀 파일은 스크립트 위치 하위의 **`output/` 폴더**에 저장됩니다(`output/Macro_Daily_DB.xlsx`). 이 폴더는 `.gitignore`로 제외되어 저장소에는 포함되지 않습니다.

## ⚠️ 5. 유의사항

- **모델 한계**: 테일러 준칙은 단순화된 규범적 모델이며, 실제 연준은 고용·금융안정·선제적 가이던스 등 다양한 요인을 종합 고려합니다. 본 도구의 결론은 **참고용 시그널**이지 매매 권고가 아닙니다.
- **데이터 지연**: GDP·CPI는 발표 시차가 있어, `ffill`로 채워진 최신 일자 값은 가장 최근 *발표된* 값 기준입니다.
- **`r*` 대용치**: 중립금리 대용으로 5년 평균 성장률을 사용한 것은 본 구현의 가정이며, 학술적 표준값(예: 라우바흐-윌리엄스 추정치)과 다를 수 있습니다.
